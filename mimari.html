Proje Disiplini

Her proje kendi dünyasıdır. BAIOS projeyi açınca şu dosyaları okur:

~/.baios/projects//
├── project_context.md      ← kalıcı: amaç, kurallar, roller
├── project_tmp_context.md  ← geçici: son işlemler, çalışma notu
├── project_config.json     ← bağlı ARM, Skill, Flow, otomasyon
├── discipline_v.md      ← LLM ile üretilmiş çalışma yöntemi
├── docs/                   ← proje materyalleri
└── outputs/                ← skill/flow çıktıları

Discipline Engine: Proje materyallerini + hedefini + mevcut BAIOS kapasitelerini birleştirir, LLM'e adım adım çalışma yöntemi ürettirir. Her versiyonu dosyaya yazar, versiyonlanır.

Proje değiştiğinde disiplin yeniden üretilebilir. Eski versiyonlar arşivde kalır.

Neden önemli:

  • LLM "BAIOS'ta şu skill yok" hatası yapmaz (liste önünde).
  • Proje değişse context kendiliğinden değişir.
  • Yeni ARM eklendiğinde tüm LLM konuşmaları otomatik bilir — stale liste yok.

ARM — Adaptör

ARM = BAIOS'u dış dünyaya bağlayan adaptör.

Kategoriler:

  • API tabanlı: Gmail, GitHub, Telegram, Notion, Brave Search, OpenWeather…
  • Yerel: shell, file, dosya_oku, web_reader, browser, sql, http, rss
  • Cloud/IoT: cloud (S3/GDrive), MQTT, HomeAssistant
  • Sistem: git_ops, deploy, feedback…
Kurulum seviyeleri (install_level):
  • 0 — Kurulumsuz, doğrudan çalışır (file, shell, web_reader)
  • 1 — API key gerekir (brave_search, weather)
  • 2 — OAuth / çok alanlı kurulum (gmail, notion, github)
  • 3 — Riskli (shell)
Her ARM:
  • Handler Python fonksiyonu: handle_(action, config, query)
  • Catalog kaydı: baios/arm/catalog.json
  • Versiyon: baios/arm/versions_local.json
  • Setup flow: sorulacak alanlar (token, repo vb.)
Yeni ARM yazmak için geliştirici rehberi.

Skill — Tekrarlanabilir İş Akışı

Skill = adım adım işleyen şablon. JSON dosyası.

{
  "name": "arastirma",
  "required_arms": ["brave_search"],
  "input_params": ["konu"],
  "steps": [
    {"action": "arm:brave_search", "params": {"query": "{konu}"}},
    {"action": "llm:Sonuçları 5 maddede Türkçe özetle."},
    {"action": "file:outputs/arastirma_{konu}.md"}
  ]
}

Adım tipleri:

  • arm: — ARM çalıştır
  • llm: — LLM ile işle (use_previous: öncekini de gönder)
  • file: — kaydet
  • skill: — başka skill çağır
Parametreler: {konu}, {hedef_kitle} gibi placeholder'lar çağrıda doldurulur.

Skill JSON'ları arkadaşlarına gönderilebilir, community store'a eklenebilir.

Flow — Çok-Skill'li Büyük Akış

Flow, birden fazla skill'i/ARM'ı koşullu, döngülü çalıştırır.

Örnek: site_release flow (baios.dev için) 

changelog_uret → release_notes → git_ops(branch) → git_ops(commit)
  → git_ops(push) → github(pr_create) → site_qa

Her adım koşullu olabilir: "QA başarısızsa rollback branch aç".

Otomasyon — Zaman/Tetik

Otomasyon = skill/flow'u zamanlanmış veya olay-tetikli çalıştırır.

  • **Cron:** `0 8 * * *` — her sabah 08:00
  • **Webhook:** gelen HTTP → flow tetikle
  • **Dosya değişimi:** `~/indirdiklerim/` izle, yeni PDF varsa analiz et
  • **RSS:** yeni item → özetle + e-posta
  • **MQTT:** sensör threshold → bildirim
Her otomasyon **onaylıdır** — kural-bazlı, risk düşükse onaysız çalışır.

MCP — Model Context Protocol

BAIOS hem MCP sunucusu hem MCP istemcisi.

  • **Sunucu:** Claude Desktop, VS Code gibi araçlar BAIOS'u MCP üzerinden kullanabilir — "ilker'in projesi baios-dev'de PR aç".
  • **İstemci:** BAIOS herhangi bir MCP sunucusuna (Figma, Jira, Notion API, GitHub MCP, Slack MCP…) bağlanır. Bir MCP sunucusu eklemek = BAIOS'a yeni ARM eklemek.
Detay: [MCP rehberi](kaynaklar/mcp-rehberi.html).

Güvenlik

  • **Vault:** Fernet + PBKDF2 ile API key'ler şifreli. Plaintext fallback yalnızca vault kilitliyse.
  • **Path traversal koruması:** Her dosya yolu resolve edilip base dir içinde mi kontrol edilir.
  • **Shell whitelist:** Yalnızca beyaz listedeki komutlar. `rm -rf`, `sudo` engellidir.
  • **git_ops sandbox:** Sadece izinli repo'lar. `push` opt-in, `--force` opt-in.
  • **Audit log:** Her aksiyon `~/.baios/audit/` altına kayıt. Kurumsal için tamper-proof.
  • **Input validation:** Her API endpoint Pydantic ile doğrulanır.
