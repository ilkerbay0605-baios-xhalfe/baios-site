Compliance Yaklaşımı

BAIOS'un veri, güvenlik ve denetim konusundaki duruşu. Ayrıntılı belge (sertifika, SOC2 rapor) kurumsal sözleşme kapsamında paylaşılır.

Veri Saklama

Bireysel kullanımda:

  • Tüm veri ~/.baios/ altında yerel.
  • LLM çağrıları dışında hiçbir dış servise veri gitmez.
  • Telemetri opsiyonel — baios config set telemetry.enabled false ile kapatılır.
Kurumsal kullanımda:
  • On-prem veya müşterinin kendi bulut hesabı.
  • BAIOS veri işlemez — altyapı müşterinin.
  • Kişisel veri geçiş noktaları (sadece LLM çağrı anı) kayıt altında.

LLM Provider Tercihi

  • **OpenAI / Anthropic:** Hızlı, kaliteli, ama veri US'de işleniyor.
  • **Azure OpenAI:** EU bölgesi seçilebilir, kurumsal SLA.
  • **Yerel Ollama:** Veri makinenin dışına çıkmaz.
  • **Kurumsal gateway:** Şirketin kendi LLM proxy'si.
Policy olarak provider kısıtlaması konulabilir: *"Gizlilik seviyesi X üstü proje sadece yerel LLM kullanabilir."*

KVKK Uyumu

Tasarım prensibi:

  • Data minimization — LLM'e yalnızca gerektiği kadar context gider, tüm dosya değil.
  • Purpose limitation — her ARM kullanımında amaç audit log'da.
  • Right to erasure — baios proje sil tüm proje verisini siler.
  • Data portability — baios proje export ZIP olarak çıkarır.
Belge: Kurumsal sözleşme kapsamında KVKK uyum raporu hazırlanır.

GDPR Uyumu

Aynı prensipler GDPR için de geçerli:

  • Data Processing Agreement (DPA) sağlanır.
  • EU veri bölgesi kısıtlaması opsiyonu.
  • Silme / taşınabilirlik hakları desteklenir.

SOC2

Hedef: Tip II hazırlık (Faz 13 sonrası).

Hazırlanan kontroller:

  • Access control (RBAC, MFA)
  • Audit logging (tamper-proof)
  • Change management (git commit + PR review)
  • Incident response playbook
  • Vendor management (LLM provider risk değerlendirmesi)

Güvenlik Mimarisi

Vault:

  • API key'ler Fernet + PBKDF2 ile şifreli.
  • Master password — kullanıcı veya kurumsal merkezi.
  • Vault kilitliyken plaintext fallback kapatılabilir.
Network:
  • TLS zorunlu (on-prem kurulumda self-signed veya kurumsal CA).
  • Web UI önünde reverse proxy (nginx/caddy).
  • API key olmadan endpoint yok.
İşletim:
  • Her ARM çağrısı audit'e kayıt (kim, ne zaman, hangi parametre).
  • Her dosya yazma resolve edilmiş yol kontrolü (path traversal yasak).
  • Shell ARM beyaz liste; kurumsal sürümde komple kapatılabilir.

Penetrasyon Testi

Bireysel:

  • Open source, herkes kaynak kodu inceleyebilir.
  • Security advisory: security@baios.dev (yapılandırılıyor).
Kurumsal:
  • Pilot müşterilerle beraber dış pentest planlanacak.
  • Bulgular koordineli açıklama (CVD) ile paylaşılacak.
