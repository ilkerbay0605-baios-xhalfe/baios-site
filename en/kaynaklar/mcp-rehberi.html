MCP — Model Context Protocol

🇹🇷 Türkçe

BAIOS is both an MCP server and an MCP client. What does that mean, and what is it good for?

What Is MCP?

Model Context Protocol — a protocol proposed by Anthropic that lets LLM tools do server-client communication in a standardized way.

A simple analogy:

  • Ethernet connects devices physically.
  • HTTP makes software talk over the web.
  • MCP makes LLM applications talk to tools/data over the same protocol.
Result: Whoever writes an MCP server (Slack, Jira, Figma) makes it usable by every MCP-compatible application (Claude Desktop, VS Code, BAIOS). Many MCP servers are already available, and the number grows every day.

BAIOS's MCP Role

As a server

BAIOS exposes itself like an MCP server — other tools from outside can reach BAIOS's ARMs/Skills/Flows.

Scenario:

  • From Claude Desktop you say "open a PR in İlker's baios-dev project"
  • Claude Desktop connects to BAIOS's MCP server
  • BAIOS's github ARM runs → the PR is opened

As a client

BAIOS connects to external MCP servers. When you add an MCP server, that tool automatically becomes an ARM.

Scenario:

  • You add the Figma MCP server (npx @figma/mcp-server)
  • BAIOS detects it → the figma ARM appears on its own (as api_figma)
  • You can use arm:api_figma in your skills

Adding an MCP Server

1. Install the MCP server

Each MCP server has its own documentation. Example — GitHub MCP:

npm install -g @modelcontextprotocol/server-github

2. Register it with BAIOS

Web UI: Settings → MCP Servers → Add

CLI: 

baios> /mcp ekle github-resmi "npx -y @modelcontextprotocol/server-github" stdio
  Environment variable:
  GITHUB_TOKEN=ghp_...

3. See it as an ARM

baios> /arm listele
# api_github_resmi — Auto ARM (MCP: github-resmi)

Now you can use it in your skills: 

{
  "action": "arm:api_github_resmi",
  "params": {"query": "list my repositories"}
}

Popular MCP Servers

Compatibility testing is ongoing. The set fully tested at 100% is currently limited; **try the ones below with confidence**:
  • **filesystem** — local file operations (a complement to BAIOS's own `file` ARM)
  • **github** — GitHub API
  • **slack** — Slack workspace
  • **google-drive** — GDrive
  • **postgres** — PostgreSQL
  • **brave-search** — web search
  • **puppeteer** — web scraping
Up-to-date list: [modelcontextprotocol.io/servers](https://modelcontextprotocol.io)

Use BAIOS From Outside

Add BAIOS to Claude Desktop as an MCP server:

1. Start BAIOS MCP server mode: 

baios mcp-server --port 7777

2. In Claude Desktop config: 

{
  "mcpServers": {
    "baios": {
      "command": "baios",
      "args": ["mcp-server", "--stdio"]
    }
  }
}

3. In Claude Desktop: 

Can you summarize the last 10 commits in the baios-dev project for me?
→ Claude uses BAIOS's git_ops ARM

Security

  • MCP servers run **locally** (stdio) or over TLS.
  • BAIOS routes MCP calls through the same approval chain (risky operation → approval).
  • The audit log records both the request coming from MCP and BAIOS's response.
  • A malicious MCP server? On the BAIOS side, **only permitted actions** get through.

Contributing

If you wrote your own MCP server and want a BAIOS compatibility test:

  • GitHub Issue: open it with the [mcp-compat] label
  • Share an example configuration + a few test commands
  • We add the test result to the repo
