TL;DR: From installation to your first project, from your first skill to your first automation, step by step.

Tutorial

The BAIOS guide from start to finish. Complete it in 45 minutes and you will have set up your own workflow.

🇹🇷 Türkçe

0. Before You Start

Requirements:

  • Python 3.12+
  • An LLM provider (API key OR local Ollama)
  • 15 minutes of focus
Result: You will become a BAIOS user with your own project, connected ARMs, and a working skill.

Part 1 — Installation (5 min)

1.1 Install BAIOS

# early version — from GitHub (PyPI release coming soon)
git clone https://github.com/ilkerbay0605-baios-xhalfe/baios
cd baios && ./run.sh start

On first launch it asks for personalization:

  • Name
  • Language (Turkish recommended)
  • Your preferred communication tone (friendly / formal / technical)

1.2 Choose an LLM provider

Local (free, no internet needed): 

# New terminal:
ollama serve
ollama pull llama3.2

# BAIOS terminal:
baios config set llm.provider local

OpenAI (speed + quality): 

baios config set llm.provider openai
baios config set openai.api_key sk-...

Anthropic (long context): 

baios config set llm.provider anthropic
baios config set anthropic.api_key sk-ant-...

1.3 Test it

baios> merhaba

A conversation should start. For the web UI: 

baios serve
# → http://localhost:8000

Part 2 — First Project (5 min)

2.1 Create a project

Think of a project tied to a purpose. For example "weekly budget tracking" or "math lessons" or "blog post pool".

baios> /proje yeni butce "Aylık gelir-gider takibi"
baios> /proje sec butce

Or from the web UI: Projects → New.

2.2 Project files

The project directory was created automatically: 

~/.baios/projects/butce/
├── project_context.md       ← purpose, rules
├── project_tmp_context.md   ← temporary notes
├── project_config.json      ← connected ARMs/Skills
├── docs/                    ← put your materials here
└── outputs/                 ← skill outputs

2.3 Add material (optional)

If you have a budget Excel: 

cp ~/bütçe.csv ~/.baios/projects/butce/docs/

2.4 Generate discipline

Web UI: Project page → Work Discipline → Write a goal → Generate

CLI: 

baios> /disiplin uret "Aylık bütçe tutmak ve ay sonu özet çıkarmak"

The result is discipline_v1.md — the LLM prepared a workflow tailored to you.

Part 3 — First ARM Setup (5 min)

3.1 What do you need?

Think: which external service does your project need?

  • Read Gmail? → gmail ARM
  • Pull info from the web? → brave_search ARM
  • Weather? → weather ARM
  • Just work with files? → no ARM needed (the file ARM is automatically available)

3.2 Install an ARM

Web UI: Library → ARM tab → find it → Install

CLI: 

baios> /arm kur weather
# → asks for an API key → get one free from OpenWeatherMap: https://openweathermap.org/api

3.3 Connect it to the project

baios> /proje bagla arm weather

Now the weather ARM is available in the project context — the LLM knows about it.

Part 4 — Running Your First Skill (10 min)

4.1 Choose a skill

arastirma is a simple and nice one to start with:

  • You provide a topic
  • It searches the web
  • The LLM summarizes
  • It saves to a file

4.2 Run it

Web UI: Library → Skill → arastirma → Run

  • konu: something that interests you, e.g. "which plants grow in the kitchen"
CLI: 
baios> /skill calistir arastirma konu="hangi bitkiler mutfakta yetişir"

4.3 Inspect the result

The output is in the project directory: 

~/.baios/projects/butce/outputs/arastirma_hangi_bitkiler_mutfakta_yetisir.md

In the web UI you can open it from Projects → butce → Files.

Part 5 — Natural Language Conversation (5 min)

Now stop memorizing commands and talk naturally:

baios> istanbul hava nasıl bugün?
baios> bütçe CSV'sinden toplam harcamayı hesapla
baios> "yeni teklif" ile başlayan mailleri listele (gmail ARM kuruluysa)
baios> bu araştırma dosyasından 5 madde özet çıkar

BAIOS reads the intent, selects the right ARM/Skill, and asks for approval on a risky operation.

Part 6 — First Automation (10 min)

Automate a repeated task.

6.1 Define an automation

baios> /otomasyon kur
  Trigger: every Sunday 20:00
  Action: produce a budget summary + email it (to me)

6.2 See it run

baios> /otomasyon listele

It runs automatically the first Sunday at 20:00. You receive the output by email.

6.3 Change / stop

baios> /otomasyon durdur butce_ozet_haftalik
baios> /otomasyon sil butce_ozet_haftalik

Part 7 — Wiki (5 min)

Make the knowledge accumulating in the project permanent:

baios> /wiki olustur butce-kategoriler

BAIOS reviews the project contents + conversation history and prepares a wiki page. On every LLM call, the relevant pages automatically go into the context.

The accumulation builds up — the older your project gets, the smarter it becomes.

Troubleshooting

Skill error "Missing ARM": install it with /arm kur .

LLM not responding: baios config show → is the provider correct, is the key valid.

File not found: Is a project selected? /proje durum.

Port conflict: baios serve --port 8080.

Log: tail -f ~/.baios/logs/baios.log
