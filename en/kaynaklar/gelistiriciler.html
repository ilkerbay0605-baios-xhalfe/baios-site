Write a New Skill

1. Create a JSON file:

baios/skills/templates/ornek_skill.json

{
  "name": "ornek_skill",
  "label": "Örnek Skill",
  "description": "Konuyu araştırır + özetler + dosyaya kaydeder.",
  "category": "arastirma",
  "required_arms": ["brave_search"],
  "input_params": ["konu"],
  "steps": [
    {
      "action": "arm:brave_search",
      "params": {"query": "{konu}"},
      "description": "Web arama"
    },
    {
      "action": "llm:Sonuçları 5 maddede Türkçe özetle.",
      "use_previous": true,
      "description": "LLM özet"
    },
    {
      "action": "file:outputs/ornek_{konu}.md",
      "use_previous": true,
      "description": "Kaydet"
    }
  ]
}

2. Test it:

python -m pytest tests/test_site_qa_skill.py -v  # structural test example
curl -X POST http://localhost:8000/api/skills/ornek_skill/run \
     -H 'Content-Type: application/json' \
     -d '{"params":{"konu":"python test patterns"}, "project":"denemem"}'

3. Update the docs:

  • Add to the skill list in agents/EZME_KORUMA.md
  • Update the skill count in PROJE_PLANI.md
Step types:
  • arm: — run an ARM (config comes from the ARM's own setup)
  • llm: — process with the LLM (use use_previous: true to append the previous output to the prompt)
  • file: — save (if the path is in a project context it goes to project/outputs)
  • skill: — call another skill
Parameters: placeholders like {konu}. They are cleaned up in file names (slugify).

Write a New ARM

Fast path: Builder endpoint (SM1.5):

curl -X POST http://localhost:8000/api/arms/create \
     -H 'Content-Type: application/json' \
     -d '{
       "name": "my_arm",
       "label": "My ARM",
       "description": "My custom ARM",
       "type": "local",
       "install_level": 0
     }'

This does three things:

  • Creates a baios/arm/my_arm_arm.py stub handler
  • Adds a record to baios/arm/catalog.json
  • Adds a register block to baios/arm/handlers.py
  • Registers it in the live _HANDLERS dict (no restart needed)
Fill in the stub:

# baios/arm/my_arm_arm.py

def handle_my_arm(action: str, config: dict, query: str):
    from baios.arm.handlers import HandlerResult
    
    # action: "search", "create", "list" etc. — determined in the skill template
    # config: contents of ~/.baios/arm_configs/my_arm.json
    # query: user question
    
    if action == "search":
        result = f"Aradığın: {query}"
        return HandlerResult(True, result)
    
    return HandlerResult(False, "", f"Bilinmeyen aksiyon: {action}")

Write a test:

# tests/test_my_arm.py
def test_my_arm_search():
    from baios.arm.handlers import execute_arm
    r = execute_arm("my_arm", "search", {}, "test")
    assert r.success is True

Write a New Flow

A flow runs multiple skills/ARMs conditionally.

baios/skills/flows/my_flow.json

{
  "name": "my_flow",
  "description": "Önce araştır, sonra blog yaz, sonra QA yap",
  "steps": [
    {
      "id": "s1",
      "type": "skill",
      "skill_name": "arastirma",
      "params": {"konu": "{konu}"},
      "save_as": "rapor"
    },
    {
      "id": "s2",
      "type": "skill",
      "skill_name": "blog_post_taslak",
      "params": {"konu": "{konu}", "okuyucu": "genel"},
      "save_as": "blog"
    }
  ]
}

Conditional step:

{
  "id": "kontrol",
  "type": "condition",
  "condition": "Blog taslak kalitesi yeterli mi? {blog}",
  "if_true": "yayinla",
  "if_false": "yeniden_yaz"
}

Change Management (P12)

Before every commit:

python3 scripts/impact_check.py      # impact of changed files
python3 scripts/drift_check.py       # doc consistency
python -m pytest tests/ -q           # full test run

Critical rules (agents/DEGISIKLIK_YONETIMI.md):

  • ARM added → update the count in 4 test files + EZME + PLAN
  • Skill added → test_e2e_full_flow + EZME
  • Endpoint added → Pydantic validation required + test

Opening a PR

1. Branch: feat/skill-ornek or fix/arm-bug-123 2. Commit format: feat: — (CycleID is in BKS) 3. python3 scripts/drift_check.py must be clean 4. Tests: 0 fail required (flake is an exception, record it in agents/cycle_active.md) 5. Open the PR: gh pr create or GitHub UI 6. Maintainer review 7. Merge
