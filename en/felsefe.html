Philosophy

🇹🇷 Türkçe

Five principles that describe not what BAIOS is, but how it behaves.

1. Language First

You write or say what you want. BAIOS understands and does it.

No commands to memorize — natural communication is the goal. Shortcuts (/projects, /skill ...) exist but are not required. You can do the same thing by writing "email Ahmet" or with the /mail send Ahmet command. BAIOS turns both into the same working language.

2. No Action Without Approval

No risky operation happens without your approval.

  • Deleting or overwriting files
  • Deploy, build, triggering a remote server
  • API calls (paid ones especially)
  • Commit, push, PR merge
Before every operation a **proposal** is shown; you approve, BAIOS acts, the result is recorded. In approved automations (pull a report every morning, back up every evening), if the risk level is low it does not ask again — **decided once, runs by the rules**.

3. Simplicity

  • Solve instead of asking the user. Take the information from context.
  • Give a 5-line answer, not 50 lines of markdown.
  • Risk tables, semantic boundaries, proposal formats — these are used **only for genuinely risky** operations (deleting files, deploy). Do not use them for information questions.
  • Do not write "no actionable step." If there is no action, move on quietly.
  • Don't offer A/B/C choices unless truly necessary — suggest the best one directly.

4. Context Is Everything

BAIOS does not depend on what the model you're talking to happens to know. BAIOS builds the context on every conversation and injects it into the model.

  • **System layer:** BAIOS version, available ARMs, skills, flows, user identity.
  • **Project layer:** Active project, project files, project discipline, connected components.
  • **Temporary layer:** Recent operations, the last five minutes of work.
  • **Wiki layer:** Knowledge-base pages relevant to the query.
Even if the model changes, the context is preserved — **even switching from OpenAI to Ollama, the conversation continues where it left off**.

5. Model Independence

BAIOS works with any model: OpenAI, Anthropic, local Ollama, MiniMax, others.

  • The model is only a **tool**. BAIOS's structure is independent of it.
  • If a provider fails, the **fallback chain** automatically moves to the next.
  • Cost-sensitive? Cheap model. Sensitive data? Local — the choice is yours.
  • One command: `baios config set llm.provider ollama`

A Layer, Not a Black Box

BAIOS does not say "leave it to me and it'll get done." It is a discipline that works beside you. You think; BAIOS produces a recorded, repeatable, auditable version of your thinking.

Why this difference matters:

  • When something acts in your place and errs, it's hard to undo.
  • When a layer errs, it can be undone step by step.
  • A black box hides its reasoning.
  • A layer is transparent — you see every step, and you can object to every step.
