Vision

BAIOS is not a PC-only layer. A sensor device squeezed into an ESP32, a robot on a Raspberry Pi, a smart-home controller — all of them speaking with the same working discipline.

  • **Natural language:** "Open the window if the living-room temperature goes above 25 degrees."
  • **Context:** The device knows its own sensors, past values and rule set.
  • **Independent:** All LLM calls run locally (small model) or via conditional cloud.
  • **Controlled:** Risky actions (start a motor, open a door) go through an approval chain.

Mini-BAIOS Profile

Architecture: A subset of full BAIOS — core + a limited ARM set.

Targeted resources:

  • ESP32 class: 8MB flash, 512KB RAM — rule engine + MQTT bridge only, LLM off-device.
  • Raspberry Pi 4 class: 1GB RAM — core + local 4K LLM + 6-8 ARMs.
  • Jetson Nano / Orin class: Full BAIOS + large local LLM.
Selected ARMs (planned):
  • mqtt — sensor read/write
  • homeassistant — smart-home REST
  • http — general
  • shell (restricted whitelist)
  • file (specific directories only)
  • sql (SQLite sensor logging)
  • rss (optional)

Use Cases

Smart-Home Control

Raspberry Pi + Home Assistant bridge. Language-based rules like "When the alarm sounds at 7 AM, open the curtains and start the coffee machine."

Sensor Data Accumulation

ESP32 cluster → MQTT → Mini-BAIOS → SQLite → daily summary LLM. Every morning, a "yesterday's average humidity/temperature/noise level for the house" report.

Robot Command Bridge

Robot operating system + BAIOS. The operator gives the command "clean the bottom-left corner of the floor," BAIOS translates the command into ROS messages and performs a risk assessment.

Maker & Hobby

ESP32 + OLED screen + sensor + BAIOS Mini. Every device at home gets an "explanation" — ask the device, the device answers.

What Is Scoped Down, What Is Not

NOT in Edge BAIOS:

  • Web UI
  • 17 channels (most are heavy)
  • Full skill/flow engine
  • Browser (Playwright)
  • Wiki system (may be restricted)
IN Edge BAIOS:
  • Project + context (light)
  • Selected ARMs
  • Rule engine
  • Approved automation
  • MQTT pub/sub integration
  • Optional local LLM (small model)
