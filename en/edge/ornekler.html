🏠 Smart Home: "Morning Routine"

Hardware: Raspberry Pi 4, Home Assistant, Zigbee bridge

Setup:

  • Project: ev
  • ARM: homeassistant, calendar, mqtt
  • Automation: "Morning alarm" + "evening lock-down"
Rule (natural language): 
every weekday at 7:00 AM:
  1. open the bedroom curtain
  2. start the coffee machine (if the water level is above 30%)
  3. tell me today's weather and calendar summary

How BAIOS does it:

  • Cron triggers → homeassistant (curtain) → homeassistant (coffee + sensor) → weather + calendar → TTS
Why BAIOS: Everything from one place, in the same language. If it breaks, change the command, don't write code.

📊 Sensor Data: "Home Health Summary"

Hardware: 5× ESP32 sensors (temperature, humidity, sound, CO2, light) + Raspberry Pi 4 brain

Setup:

  • The ESP32s publish to MQTT (values per second)
  • Mini-BAIOS on the Pi listens to MQTT
  • Logs to SQLite
  • Every morning at 8:00 a daily summary goes to the LLM
Sample report:
"Last night the CO2 level in the living room rose to 1400 ppm (03:00-05:00). In the bedroom, noise stayed above 45dB for 2 hours. My suggestion: open the ventilation for 15 min at 22:00 in the evening."

BAIOS skill: sensor_gun_ozet (custom for edge)

🤖 Robot: "Room-Cleaning Commands"

Hardware: Jetson Orin Nano + LIDAR + ROS 2

Setup:

  • The operator gives a voice command
  • BAIOS STT → intent analysis → ROS 2 goal message
  • Risk check (is there anything breakable, is the charge enough)
  • Action → logged to audit
Sample command: 
operator: "start from the right side of the kitchen, move to the living room within 20 minutes"
BAIOS: 
  - rota_planla(start="mutfak_sag", end="salon", sure=20dk)
  - charge=85% → OK
  - breakable_item detected → "caution: vase x=3.2 y=1.8"
  - approve? → operator OK
  - ROS goal publish

🔧 Maker: "Talking Aquarium"

Hardware: ESP32 + OLED + temperature/pH sensor + Mini-BAIOS (on a Pi Zero)

Setup:

  • Reads the aquarium sensors
  • The Mini-BAIOS on the Pi Zero knows the species of the user's fish and their care rules
  • Email/Telegram on anomaly
User chat (from the phone): 
user: "how's the water today?"
BAIOS: "Temperature 24.8°C (ideal 24-26), pH 7.2 (ideal 7.0-7.5). 
       The nitrite level rose to 0.1 yesterday, today it's 0 — good. 
       You haven't changed the feed for 3 days, shall I remind you?"

🚗 Driver Assistant (in-vehicle)

Hardware: Jetson Nano + CAN-bus adapter + microphone + speaker

Setup:

  • Access to vehicle sensors (fuel, engine temperature, speed)
  • BAIOS talks to the driver in natural language
  • Traffic data (offline map data) + weather
Scenario: 
user: "is there enough fuel to get home?"
BAIOS: "Yes, ~280 km with the remaining fuel. Home is 95 km away. 
       There are 2 gas stations on the way, no need. 
       The weather is turning — rain starts in 35 min, take care."

⚙️ Industrial: Machine Maintenance Prediction

Hardware: Factory machine PLC + Modbus gateway + Raspberry Pi cluster + Mini-BAIOS

Setup:

  • PLC sensor data → MQTT
  • BAIOS hourly anomaly detection (simple ML)
  • Telegram notification + file log to the maintenance engineer
User: 
engineer: "how has machine 3 been over the last week?"
BAIOS: "Vibration rose 18% in week 3. In the last 24 hours the local peak 
       is 32%. If this trend continues, the bearing will need 
       replacing within 9-12 days. Shall I check if there's a spare in stock?"

Common Feature: All the Same BAIOS

In all six scenarios the same core runs:

  • Project discipline
  • Context layers
  • Approval chain
  • Audit log
The only difference is which ARMs are active and where the LLM runs (local vs cloud).
